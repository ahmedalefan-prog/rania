## 2. العقبة التقنية الأساسية: ويندوز ↔ آيفون

برمجة تطبيق آيفون **أصلي** تتطلب جهاز Mac وبرنامج Xcode. جهازك ويندوز، لذا أمامنا 3 مسارات:

### المسار (أ) — تطبيق ويب تقدمي PWA  ✅ (توصيتي)
- يُبنى بالكامل على ويندوز.
- يُفتح من متصفح Safari في الآيفون، ثم **"إضافة إلى الشاشة الرئيسية"** فيظهر كأيقونة تطبيق عادي.
- **يعمل بدون إنترنت** بعد أول فتح (تخزين محلي على الهاتف).
- لا يحتاج Mac، ولا حساب مطوّر Apple، ولا رسوم سنوية.
- **العيب:** ليس داخل App Store، وتخزين iOS يحتاج نسخاً احتياطية منتظمة (سنعالجها بقوة في التصميم).

### المسار (ب) — تطبيق أصلي عبر Expo / React Native
- تطبيق "حقيقي" بإحساس أصلي، يمكن تجربته على الآيفون عبر تطبيق Expo Go.
- يمكن بناء نسخة iOS عبر خدمة سحابية (EAS) دون امتلاك Mac.
- **العيب:** لنشره في App Store يلزم حساب Apple Developer (99$ سنوياً) + خطوات أعقد.

### المسار (ج) — تطبيق أصلي Swift/SwiftUI
- الأفضل أداءً وتكاملاً مع iOS.
- **العيب:** **مستحيل** بدون جهاز Mac. غير مناسب لوضعك الحالي.

> **توصيتي:** نبدأ بالمسار (أ) PWA. لطبيبة واحدة، عيادة واحدة، بدون إنترنت — هذا أسرع وأرخص وأكثر عملية، ويغطي 100% من الميزات المطلوبة. ويمكن لاحقاً تحويله لتطبيق App Store إن أردتِ التوسع.

## 3. البنية التقنية المقترحة (للمسار أ)

| الطبقة | التقنية | السبب |
|---|---|---|
| الإطار | Next.js (React) + TypeScript | حديث، مستقر، يدعم PWA |
| الواجهة | Tailwind CSS + shadcn/ui | تصميم احترافي سريع |
| دعم العربية | RTL كامل (يمين ← يسار) | الواجهة بالعربية |
| قاعدة البيانات | Dexie.js على IndexedDB | تخزين محلي على الهاتف، يعمل أوفلاين |
| العمل أوفلاين | Service Worker (Serwist) | يعمل بدون شبكة |
| الرسوم البيانية | Recharts | تقارير الإيرادات والمصاريف |
| مخطط الأسنان | مكوّن SVG مخصّص | تفاعلي بالأسطح الخمسة |
| النسخ الاحتياطي | تصدير/استيراد ملف JSON + iCloud Files | حماية البيانات الطبية |

## 4. نموذج البيانات (الجداول)

- **Patient** (المريض): الاسم، العمر/تاريخ الميلاد، الهاتف، الأمراض المزمنة، الحساسية، ملاحظات، الرصيد المتبقي (محسوب تلقائياً).
- **ToothCondition** (حالة السن): رقم المريض، رقم السن (FDI)، السطح (Mesial/Distal/Occlusal/Buccal/Lingual أو كامل)، نوع الإجراء، الحالة (مخطط/مكتمل)، التاريخ، التكلفة، ملاحظة.
- **Appointment** (الموعد): المريض، التاريخ والوقت، السبب، الحالة (قادم/تم/ألغي/لم يحضر).
- **Transaction** (حركة مالية): النوع (إيراد/مصروف)، الفئة (علاج/راتب/إيجار/كهرباء/مشتريات...)، المبلغ، التاريخ، الوصف، ربط بمريض إن وُجد.
- **Employee** (الموظف): الاسم، الوظيفة، الراتب، أيام الدوام/العطلة، الحضور.
- **Settings**: العملة، اسم العيادة، الشعار، أسعار الإجراءات الافتراضية.

## 5. تصميم مخطط الأسنان (Odontogram)

- **ترقيم FDI الدولي** كما اقترحت (صحيح تماماً).
- **زر تبديل** بين أسنان البالغين (32) وأسنان الأطفال اللبنية (20، ترقيم 55–85).
- **كل سن مقسّم إلى أسطح** (5 للأضراس، والأمامية تستخدم Incisal بدل Occlusal) — كما في البرامج الاحترافية.
- **طبقتان لكل سن:**
  1. **الحالة الموجودة** (ما تم فعلاً).
  2. **خطة العلاج المقترحة** (قبل التنفيذ) ← تُعرض على المريض ثم تُحوّل إلى "مكتمل".
- **التلوين الذكي** حسب الإجراء (علاج عصب 🟡، تلبيسة 🟢، زراعة 🔵، قلع 🔴، تقويم 🟣، سليم ⚪، تسوس بني...).
- قائمة الإجراءات عند الضغط على السن: فحص، تسوس، حشوة، حشوة تجميلية، علاج عصب، وتد، تلبيسة، جسر، زراعة، قلع، تقويم، تنظيف، تبييض، أخرى.
- **ربط تلقائي بالمحاسبة:** كل إجراء بتكلفة ← يُضاف لفاتورة المريض ← يُحدّث الرصيد تلقائياً.

## 6. الوحدات والمراحل

### المرحلة 1 — النواة (MVP)
1. ملف المريض (الـ3 تبويبات: الملف الشخصي / المخطط / السجل الزمني).
2. مخطط الأسنان التفاعلي بالأسطح الخمسة والتلوين.
3. السجل الزمني للإجراءات.
4. حفظ محلي يعمل أوفلاين + نسخة احتياطية (تصدير/استيراد).

### المرحلة 2 — الإدارة
5. جدولة المواعيد (تقويم يومي/أسبوعي).
6. المحاسبة: إيرادات + مصاريف (رواتب، إيجار، كهرباء، مشتريات) + تقارير.

### المرحلة 3 — الموظفون والتقارير المتقدمة
7. إدارة الموظفين (الدوام، العطلة، الراتب، الحضور).
8. لوحة تحكم رئيسية (Dashboard) بمؤشرات العيادة.
9. تحسينات الطباعة والفواتير.

---

## 7. الخصوصية والنسخ الاحتياطي (مهم جداً للبيانات الطبية)

- كل البيانات تُخزّن **محلياً على هاتف د. رانيا فقط** — لا ترسَل لأي خادم.
- **خطر:** فقدان الهاتف = فقدان البيانات. لذلك:
  - زر **"نسخة احتياطية"** يصدّر كل البيانات كملف يُحفظ في iCloud / Files / يُرسل للبريد.
  - زر **"استعادة"** من ملف نسخة احتياطية.
  - تذكير دوري بأخذ نسخة احتياطية.
- إمكانية **قفل التطبيق برمز/بصمة (Face ID)**.

| المرحلة | المخرجات | التقدير التقريبي |
